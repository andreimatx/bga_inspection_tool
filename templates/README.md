# BGA Inspection Report Templates

These files are generated by `tools/generate_report_templates.py` using the
**same code** that writes real inspection reports, so they always match the
production format. The sample data contains 8 balls covering every
classification band (acceptable / process indicator / defect /
grid-estimated) so all formatting rules are demonstrated — that is why the
template verdict shows REJECT.

Regenerate after any reporting change:

```
python tools/generate_report_templates.py
```

## Files produced per inspected board

| File | Purpose |
|---|---|
| `<board>_ball_metrics.csv` | Machine-readable per-ball table |
| `<board>_summary.csv` | Machine-readable one-line board summary |
| `<board>_inspection_report.xlsx` | Presentation-grade Excel report (4 sheets) |
| `<board>_inspection_report.docx` | Printable Word report with charts and X-ray preview |

## Excel sheets

1. **Executive Summary** — verdict banner, inspection details, key metrics,
   void-ratio distribution with charts.
2. **IPC Evaluation** — every acceptance criterion with its standard source,
   limit, measured value and PASS/INFO/FAIL result.
3. **Ball Details** — full per-ball table; red rows = defect balls, amber
   rows = process indicators, color scale on total void %.
4. **Algorithm Analysis** — honest assessment: how the numbers were
   produced, measurement uncertainty, known limitations, and the list of
   balls to verify manually first.

## Acceptance criteria (see `src/standards.py`)

| Rule | Source | Effect |
|---|---|---|
| Cumulative void area per ball > **25%** | IPC-A-610 (Class 1, 2, 3) | **DEFECT / FAIL** |
| Total void in the **10–25%** band | IPC-7095 process guidance | PROCESS INDICATOR / REVIEW |
| Largest single void ≥ **12.25%** area (~35% of ball diameter) | IPC-7095 dominant-void guidance | PROCESS INDICATOR / REVIEW |
| Voids at pad interface (10% / 30% dia limits) | IPC-7095 | **NOT EVALUATED** — impossible from a single top-down 2D X-ray; stated honestly instead of guessed |

## Per-ball CSV columns

| Column | Meaning |
|---|---|
| `ball_id` | Sequential ball number (row-major over the grid) |
| `center_x`, `center_y` | Ball center in image pixels |
| `diameter_px`, `ball_area_px` | Detected ball size (projected) |
| `void_count` | Number of accepted void components |
| `total_void_area_px` | Sum of accepted void areas (px²) |
| `largest_void_area_px` | Area of the biggest single void (px²) |
| `void_ratio_percent` | Total void area as % of ball area — the IPC-A-610 metric |
| `largest_void_ratio_percent` | Largest single void as % of ball area |
| `largest_void_diameter_percent` | Largest void equivalent diameter as % of ball diameter (IPC-7095 wording) |
| `quality` | PASS / REVIEW / FAIL (see criteria above) |
| `ipc_assessment` | Same, in report wording: ACCEPTABLE / PROCESS INDICATOR / DEFECT (IPC-A-610) |
| `is_estimated_pad` | Pad placed from grid geometry without direct image evidence — treat metrics with care |
| `confidence` | Detection confidence of the underlying circle |
